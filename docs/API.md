# CivicOS â€” REST + WebSocket API Reference

CivicOS is a single FastAPI application hosting four civic modules behind a shared
authentication layer:

| Module | Name | Prefixes |
|--------|------|----------|
| 1 | Digital Queue System | `/institutions`, `/tickets`, `/ws/queue/...`, `/admin/queue/*` |
| 2 | Document Tracker | `/files`, `/document-types`, `/ws/files/...`, `/admin/files/*` |
| 3 | Local Market | `/listings`, `/sellers/...` |
| 4 | Urban Issue Reporter | `/issues`, `/issue-categories`, `/admin/issues/*` |

All paths are mounted at the application root (the API uses no `/v1` prefix â€”
`api_v1_prefix` defaults to `""`).

## Interactive documentation

These are auto-generated by FastAPI and always reflect the live code:

| URL | Description |
|-----|-------------|
| `GET /docs` | Swagger UI (try-it-out console) |
| `GET /redoc` | ReDoc reference |
| `GET /openapi.json` | Raw OpenAPI 3 schema |

## Authentication model

Two token types, both JWT (HS256), default lifetime **30 days**
(`jwt_expire_days`). Send as a bearer token:

```
Authorization: Bearer <access_token>
```

| Auth label used below | Meaning |
|-----------------------|---------|
| **public** | No token required. |
| **citizen JWT** | Token with `role=citizen`, obtained from `/auth/verify-otp`. `sub` = phone. |
| **admin JWT** | Token with `role=admin`, obtained from `/admin/login`. `sub` = username; payload also carries `admin_id`, `institution_id`, `institution_type`, `is_superuser`. |

Notes:
- Missing token â†’ `401 {"detail": "Missing bearer token"}`. Invalid/expired â†’
  `401 {"detail": "Invalid or expired token"}`.
- Wrong role â†’ `403` (`"Citizen token required"` / `"Admin token required"`).
- Non-superuser admins are scoped to their own `institution_id` (cross-institution
  access returns `403`).
- WebSocket endpoints are **public** (no token).

# Auth

### `POST /auth/request-otp`
- Auth: public
- Description: Generate and "send" a one-time code for a phone number. In
  development the code is printed to the server console and (when
  `otp_debug_return` is on, the default) echoed back as `debug_otp`. Keep
  `otp_debug_return=false` in production so `debug_otp` is `null`.

Request:
```json
{ "phone": "+21655123456" }
```

Response `200`:
```json
{
  "message": "OTP sent to +21655123456",
  "debug_otp": "123456"
}
```

### `POST /auth/verify-otp`
- Auth: public
- Description: Exchange a valid OTP for a citizen JWT. Invalid/expired code â†’
  `401 {"detail": "Invalid or expired OTP"}`.

Request:
```json
{ "phone": "+21655123456", "otp": "123456" }
```

Response `200`:
```json
{
  "access_token": "eyJhbGciOi...",
  "token_type": "bearer",
  "role": "citizen",
  "phone": "+21655123456",
  "expires_in_days": 30
}
```

### `POST /admin/login`
- Auth: public
- Description: Admin username/password login. Bad credentials â†’
  `401 {"detail": "Invalid username or password"}`.

Request:

Response `200`:

# Module 1 â€” Digital Queue System

## Public endpoints

### `GET /institutions`
- Auth: public
- Description: List/search institutions, each enriched with today's live load
  (waiting count, current number served, queue status, estimated wait).
- Query params: `q` (free-text on name/address), `city`, `country`,
  `type` (`hospital|municipality|post|court|tax_office`),
  `active_only` (bool, default `true`).

Response `200`:
```json
[
  {
    "id": 1,
    "name": "Tunis Central Post Office",
    "type": "post",
    "address": "Av. Habib Bourguiba",
    "city": "Tunis",
    "country": "Tunisia",
    "avg_wait_minutes": 8,
    "is_active": true,
    "waiting_count": 12,
    "current_number": 47,
    "queue_status": "open",
    "estimated_wait_minutes": 104
  }
]
```

### `GET /institutions/{institution_id}/today`
- Auth: public
- Description: Today's full queue state for one institution. `404` if the
  institution does not exist. If no queue exists yet, queue fields default
  (`status: "closed"`, counters `0`, empty lists).

Response `200`:
```json
{
  "institution": {
    "id": 1, "name": "Tunis Central Post Office", "type": "post",
    "address": "Av. Habib Bourguiba", "city": "Tunis", "country": "Tunisia",
    "avg_wait_minutes": 8, "is_active": true
  },
  "queue_id": 5,
  "date": "2026-06-15",
  "status": "open",
  "current_number": 47,
  "total_served": 46,
  "waiting_count": 12,
  "windows": [
    { "id": 1, "window_number": 1, "agent_name": "Window 1", "current_ticket_id": 51 }
  ],
  "next_numbers": [48, 49, 50]
}
```

### `POST /tickets`
- Auth: public
- Description: Take a ticket (auto-creates today's queue if needed). Broadcasts a
  `ticket_created` event on the queue channel. Errors: `404` institution not
  available; `409` queue closed; `409` queue paused.

Request:
```json
{ "institution_id": 1, "phone": "+21655123456", "service_type": "parcel_pickup" }
```

Response `201`:
```json
{
  "id": 51,
  "number": 48,
  "position": 12,
  "estimated_wait_minutes": 96,
  "institution_id": 1,
  "status": "waiting"
}
```

### `GET /tickets/{phone}`
- Auth: public
- Description: A citizen's tickets, most recent first (max 50).

Response `200`:
```json
[
  {
    "id": 51, "queue_id": 5, "number": 48, "phone": "+21655123456",
    "service_type": "parcel_pickup", "status": "waiting",
    "created_at": "2026-06-15T09:12:00Z", "called_at": null,
    "served_at": null, "wait_minutes": null
  }
]
```

### `DELETE /tickets/{ticket_id}`
- Auth: public
- Description: Cancel a ticket (sets status `cancelled`). Broadcasts
  `ticket_cancelled`. Errors: `404` not found; `409` if already
  `served|no_show|cancelled`. Returns the updated ticket (`TicketOut`).

Response `200`: same shape as a `TicketOut` item above, with `status: "cancelled"`.

## WebSocket

### `WS /ws/queue/{institution_id}`
- Auth: public
- Description: Subscribe to live queue updates. On connect the server pushes a
  `snapshot`; thereafter it emits an event whenever a ticket is taken, called,
  cancelled or finished, or the queue is opened/paused/closed. Inbound client
  messages are ignored (pure broadcast feed).

Initial message:
```json
{
  "event": "snapshot",
  "institution_id": 1,
  "status": "open",
  "current_number": 47,
  "waiting_count": 12,
  "next_numbers": [48, 49, 50],
  "windows": [ { "id": 1, "window_number": 1, "agent_name": "Window 1", "current_ticket_id": 51 } ]
}
```
> The payload is the queue state produced by the service layer
> (`queue_state_payload`); `event` varies per update:
> `snapshot`, `ticket_created`, `ticket_cancelled`, `ticket_called`,
> `ticket_no_show`, `ticket_served`, `queue_opened`, `queue_paused`,
> `queue_closed`, `queue_empty`.

## Admin endpoints (admin JWT)

All take an optional `institution_id` query param; if omitted, the admin's own
institution is used. Non-superusers may only operate on their own institution
(`403` otherwise).

### `GET /admin/dashboard`
- Auth: admin JWT
- Description: Today's queue state for the admin's institution (same
  `TodayQueueOut` shape as `/institutions/{id}/today`; `next_numbers` limited to 10).

### `POST /admin/queue/open`
- Auth: admin JWT
- Description: Open (or create) today's queue and ensure the requested number of
  service windows exist. Broadcasts `queue_opened`. Returns `TodayQueueOut`.
- Query param: `institution_id` (optional).

Request body:
```json
{ "windows": 3 }
```
(`windows` integer, default `1`, range 1â€“50.)

### `POST /admin/queue/pause`
- Auth: admin JWT
- Description: Pause today's queue. Broadcasts `queue_paused`. `409` if no queue.
  Returns `TodayQueueOut`.

### `POST /admin/queue/close`
- Auth: admin JWT
- Description: Close today's queue. Broadcasts `queue_closed`. `409` if no queue.
  Returns `TodayQueueOut`.

### `POST /admin/queue/next`
- Auth: admin JWT
- Description: Complete any currently-called ticket, then call the next waiting
  number. Notifies the citizen by SMS and broadcasts `ticket_called`. `409` if
  queue not open; `404 {"detail": "No one is waiting"}` (also broadcasts
  `queue_empty`).

Response `200` (`CallTicketResponse`):
```json
{
  "ticket": {
    "id": 52, "queue_id": 5, "number": 49, "phone": "+21655000000",
    "service_type": null, "status": "called",
    "created_at": "2026-06-15T09:20:00Z", "called_at": "2026-06-15T10:05:00Z",
    "served_at": null, "wait_minutes": null
  },
  "window_number": 1,
  "message": "Now serving number 49"
}
```

### `POST /admin/queue/call/{number}`
- Auth: admin JWT
- Description: Call a specific ticket number for today. Errors: `409` queue not
  open; `404` no ticket with that number; `409` if that ticket is already
  `served|cancelled|no_show`. Returns `CallTicketResponse` (as above).

### `POST /admin/tickets/{ticket_id}/no-show`
- Auth: admin JWT
- Description: Mark a ticket `no_show` and clear it from its window. Broadcasts
  `ticket_no_show`. `404` if not found. Returns `TicketOut`.

### `POST /admin/tickets/{ticket_id}/served`
- Auth: admin JWT
- Description: Finish serving a ticket without calling the next one (sets
  `served`, computes `wait_minutes`, increments `total_served`). Broadcasts
  `ticket_served`. `404` if not found. Returns `TicketOut`.

### `GET /admin/stats`
- Auth: admin JWT
- Description: Today's aggregate statistics. `404` if no queue today.
- Query param: `institution_id` (optional).

Response `200` (`QueueStatsOut`):
```json
{
  "institution_id": 1,
  "date": "2026-06-15",
  "total_tickets": 60,
  "served": 46,
  "no_show": 2,
  "cancelled": 1,
  "waiting": 11,
  "avg_wait_minutes": 7.4,
  "current_number": 47,
  "status": "open"
}
```

```json
  "expected_ready_date": "2026-06-20",
  "notes": "Urgent",
  "document_type": {

      "message": "File request submitted", "updated_by": "system",
      "updated_at": "2026-06-15T09:00:00Z"
    }
  ]
}
```

### `GET /files/phone/{phone}`
- Auth: public
- Description: All files for a phone number, newest first. Returns `list[FileOut]`
  (same as a `FileWithHistory` item but without the `updates` array).

### `GET /files/{reference}`
- Auth: public
- Description: Track a file by reference number. `404` if not found. Returns
  `FileWithHistory` (shape as in `POST /files` response).

### `GET /files/{reference}/history`
- Auth: public
- Description: The status-change history for a file. `404` if not found.

Response `200`:
```json
[

    "id": 1, "old_status": null, "new_status": "submitted",
    "message": "File request submitted", "updated_by": "system",

    "updated_at": "2026-06-16T11:30:00Z"
  }
]
```

## WebSocket

### `WS /ws/files/{reference}`
- Auth: public
- Description: Subscribe to status updates for one file. On connect the server
  pushes a `snapshot` with the current status; thereafter a `status_changed`
  event is emitted whenever an admin updates the file. Inbound messages ignored.

Initial message:
```json
{ "event": "snapshot", "reference": "REF-8K2P4Q", "status": "submitted" }
```

Update message:
```json
{
  "event": "status_changed",
  "reference": "REF-8K2P4Q",
  "old_status": "submitted",
  "status": "processing",
  "message": "Sent to records office"
}
```

## Admin endpoints (admin JWT)

Non-superuser admins only see/affect files whose document type belongs to their
own institution.

### `GET /admin/files`
- Auth: admin JWT
- Description: List files (max 500), newest first.
- Query params: `status` (filter by file status), `phone`.
- Response: `list[FileWithHistory]`.

### `PUT /admin/files/{file_id}/status`
- Auth: admin JWT
- Description: Change a file's status, append a history entry, optionally set a
  new expected date, auto-SMS the citizen, and broadcast `status_changed`.
  `404` if file not found. Returns `FileWithHistory`.

Request:
```json
{ "status": "ready", "message": "Ready for pickup at counter 4", "expected_date": "2026-06-19" }
```
(`status` is one of `submitted|processing|ready|delivered|rejected`; `message`
and `expected_date` optional.)

### `POST /admin/files/{file_id}/notify`
- Auth: admin JWT
- Description: Send the citizen an ad-hoc SMS about a file (default message
  derived from current status if `message` omitted). `404` if file not found.

Request:
```json
{ "message": "Please bring your original ID when collecting." }
```

Response `200`:
```json
{ "message": "Notification sent to +21655123456" }
```

## Admin endpoints (admin JWT)

### `GET /admin/issues`
- Auth: admin JWT
- Description: List issues for the municipality dashboard (max 1000), newest first.
- Query params: `city`, `status`, `priority`, `department`.
- Response: `list[IssueWithHistory]`.

### `PUT /admin/issues/{issue_id}/status`
- Auth: admin JWT
- Description: Update an issue's status, append a history entry (optionally with a
  `photo`), set `resolved_at`/`resolution_note` when moving to `resolved`/`closed`,
  SMS the reporter, and broadcast `issue_status_changed`. `404` if not found.
  Returns `IssueWithHistory`.

Request:
```json
{ "status": "in_progress", "message": "Crew dispatched", "photo": "https://.../work.jpg" }
```
(`status` âˆˆ `reported|acknowledged|in_progress|resolved|closed`; `message` and
`photo` optional.)

### `POST /admin/issues/{issue_id}/assign`
- Auth: admin JWT
- Description: Assign an issue to a department (optionally setting priority). If
  the issue is still `reported` it is bumped to `acknowledged`. Adds a history
  entry and broadcasts `issue_assigned`. `404` if not found. Returns `IssueOut`.

Request:
```json
{ "department": "Public Works", "priority": "urgent" }
```
